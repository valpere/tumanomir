# Sanity check: H_spec на реальних даних

Експериментальні дані для розділу «Sanity check» статті «Джерело Невідомості»
(`../SourceOfTheUnknown-Plan.md`). Дата: 2026-07-03.

## Протокол

Специфікація → Go type definitions, N=10 генерацій @ temperature=1.0 →
Go AST → фіче-вектори → попарна косинусна схожість → single-linkage
кластеризація @0.95/@0.80 → ентропія Шеннона по кластерах.
H_max(N=10) = log₂10 ≈ 3.32 біт.

Два незалежні прилади:

- **A**: `qwen3-coder:30b` (Ollama local) — `gen.sh` → `out/`
- **B**: `kimi-k2.7-code:cloud` (Ollama cloud, **think=false**) — `gen-cloud.sh` → `out-cloud/`

крос-інструментально з майже однаковою величиною: дозоване збільшення
точності входу дає вимірюваний відгук, і це не артефакт однієї моделі.

## Відтворення

```bash
./gen.sh          # прилад A → out/     (потребує ollama + qwen3-coder:30b)
./gen-cloud.sh    # прилад B → out-cloud/ (потребує ollama cloud)
cd analyze && go build -o analyze . && ./analyze ../out && ./analyze ../out-cloud
```

## Граблі, на які ми вже наступили (частина протоколу)

- **think-режим — частина приладу.** З thinking=on kimi з'їдав увесь
  num_predict на reasoning для складних специфікацій → порожній content,
  причому збій корелював зі складністю входу (систематичне спотворення).
- **num_predict має перевищувати природну довжину виходу.** Обрізана
  генерація парситься як невалідна → фальшивий «окремий кластер».
- **Невалідний Go** — регенерація до N валідних із фіксацією discard count.
- **num_ctx має вміщати всю специфікацію.** Дефолтний контекст Ollama
  (4096) менший за велику спеку — модель мовчки бачить обрізаний вхід
  і міряє не той документ.
