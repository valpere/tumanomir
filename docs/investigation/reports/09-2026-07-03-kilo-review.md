# Рецензія плану "Джерело Невідомості"

## 1. Сильні сторони

- **Концептуальна дихотомія**: чітка дихотомія між "джерелом істини" та "джерелом розподілу можливих реалізацій" — фундаментальна перспектива для ери ШІ-агентів.
- **Розмежування трьох режимів збою**: неоднозначність/неповнота/нетрасованість — практичне та теоретично обґрунтоване розглядення.
- **Концепція бюджету точності**: перехід від бінарної оцінки до континууму допусків — інженерний підхід.
- **Розмежування зон**: реалізація vs пошук — ключовий нюанс, що запобігає застосуванню методу випадково.
- **Експериментальна валідація**: реальні дані, два незалежних прилади, фіксований протокол — науковий rigor.

## 2. Слабкі місця та технічні помилки

### Формули метрик

- **D_const**: формула вимірює співвідношення структурованих звязків до текстових вузлів, але називається "щільністю обмежень". Це не той самий показник — щільність обмежень має вимірювати відношення constraint-визначень до альтернативних реалізацій.
- **K_drift**: назва "реляційний дрейф" не відповідає суті (відсутність трасування). Краще "traceability gap" або "orphaned requirements rate".
- **H_spec**: використання ентропії Шеннона на розподілі ймовірностей по кластерах є метрикою розподілу, не дисперсії. Для вимірювання дисперсії (variance) треба середньоквадратичне відхилення від середньої схожості або іншу статистику розподілу.

### Методологія

- **Baseline H=0**: це не означає нульовий шум приладу — може означати дуже монотонну модель. Шум треба оцінювати через ΔH, але варто додати bootstrap-аналіз чутливості.
- **Параметрична залежність**: формули не враховують залежність від промпту. H_spec вимірює властивість "специфікація + промпт", а не лише специфікації.
- **Статистична значущість**: відсутній аналіз чи різниця fog (3.32) vs sharp (1.77) значима при N=10. Потрібен confidence interval.

## 3. Що покращити

- Переформулювати D_const як співвідношення constraint-кількості до неstructured-вмісту, або додати другу метрику.
- Перейменувати K_drift у більш інформативне.
- Додати альтернативну метрику дисперсії: σ² = Σ(sim_i - μ)² / (n-1), де sim_i — попарна схожість.
- Нормалізувати H_spec до [0,1] замість log₂(N) для крос-модельних порівнянь.
- Додати рекомендацію щодо bootstrap-оцінки статистичної довіри.

## 4. Що додати

- **Аналіз чутливості**: як зміна порогу кластеризації (0.95 → 0.80) впливає на рейтинги.
- **Альтернативний підхід**: порівняти з formal verification (TLA+, Alloy) — чому стохастичний підхід інший.
- **Поради інтерпретації**: що робити при H=2.5 біт (наприклад), які кроки автору ТЗ.
- **Гістограми розподілу**: кількість кластерів, їх розмір, розподіл схожості — візуалізація допоможе калібрувати інтуїцію.
